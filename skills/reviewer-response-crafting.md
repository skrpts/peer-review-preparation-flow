## Capability

Crafts professional, thorough responses to peer reviewer comments for journal manuscript revisions. This skill handles the full range of reviewer feedback — from minor typographical corrections to major methodological concerns — producing point-by-point responses that are respectful, thorough, and strategically effective. It also drafts cover letters for submission and resubmission.

## When to Use

- Drafting point-by-point responses to reviewer comments after receiving a "revise and resubmit" decision
- Writing a cover letter for initial journal submission
- Writing a cover letter accompanying a revised manuscript
- Responding to editor queries or requests for additional information
- Preparing a resubmission to a different journal after rejection (explaining changes made in response to previous reviews)

## Inputs

- Reviewer comments (verbatim, from all reviewers)
- Editor's decision letter
- Original manuscript (pre-revision)
- Revised manuscript (post-revision, if changes have already been made)
- List of changes made in response to each comment (or indication that no change was made, with justification)
- For cover letters: manuscript title, abstract, key findings, target journal, editor name

## Process

1. **Categorise comments** — classify each reviewer comment by type (methodological concern, writing clarity, additional analysis requested, missing reference, factual correction, formatting issue, editorial suggestion) and severity (major, minor, editorial)
2. **Draft individual responses** — for each comment, produce a response that:
   - Begins by thanking the reviewer for the specific feedback
   - Restates the concern in the author's own words (demonstrating understanding)
   - Describes the action taken (change made, analysis added, text revised) with specific references to page numbers and line numbers in the revised manuscript
   - If no change was made: provides a clear, evidence-based justification
   - Maintains a respectful, collegial tone throughout, even for unfair or poorly informed comments
3. **Handle contradictory feedback** — when reviewers disagree with each other, acknowledge both perspectives, explain how you resolved the tension, and invite the editor's guidance
4. **Draft cover letter** — for initial submissions: highlight novelty, contribution, and fit. For revisions: summarise the main changes made and express gratitude for the reviewers' time
5. **Cross-reference changes** — verify that every claimed change in the response letter is actually present in the revised manuscript. Flag any discrepancies

## Outputs

- Complete point-by-point response document organised by reviewer
- Cover letter for the editor
- Summary of changes table (change number, reviewer comment, action taken, location in revised manuscript)
- List of any unresolved issues or areas where editorial guidance is sought

## Constraints

- Never be dismissive or hostile, even when reviewer comments are wrong or unfair — the tone must always be professional and grateful
- Never agree to make a change that would misrepresent the research or introduce errors
- When declining to make a requested change, provide a substantive, evidence-based justification — not just "we disagree"
- Do not pad responses with unnecessary verbiage — reviewers and editors appreciate concise, direct responses
- Always address every single comment, no matter how minor — skipping comments signals carelessness
- Clearly distinguish between changes to the manuscript and explanations provided only in the response letter
